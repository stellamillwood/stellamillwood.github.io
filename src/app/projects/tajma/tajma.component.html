Tajma is a task-based time reporting tool used daily by 11,000+ internal users at the Swedish Public Employment Office. It is built as an Angular web application maintained by me and a full-stack colleague. I owned the front-end and UX design responsibilities—ensuring a stable interface with an intuitive and efficient user experience.

Highlights

  • Migrated the front end to the internal design system, improving layout consistency, accessibility, and long-term code sustainability.
  • Improved table accessibility for visually impaired users through iterative user testing and close collaboration with accessibility specialists.

Challenges

When I joined, the application had some aesthetic limitations, did not meet accessibility standards, and the front-end codebase was rigid to work in. The team also received a large volume of reports about accidental week “deliveries” (which disabled editing) and requests for easier access to task descriptions previously offered in a discontinued “day” view.

In this view users cannot edit time input because the week has already been "delivered" to the economy department.

Design System Migration

Migrating to the internal design system involved occasional technical gaps. I contributed improvements directly to the open-source components and collaborated with the design system developers on unresolved issues. I later demoed the results for ~150 colleagues, highlighting the design system’s functionality and potential, which increased awareness and confidence in its adoption.

Task Descriptions and Error Prevention

Reports about missing task descriptions were few and sometimes indirect, so I conducted five user tests and interviews to understand the underlying workflow. This led to a collapsible task description in the table, reducing time spent switching contexts.

To address accidental week “deliveries,” I first adjusted copy to clarify consequences (“Deliver week” → “Lock week”) and added explanatory text. Reports decreased significantly. After aligning with the Economy department on requirements, we removed the feature entirely and communicated the change to users.

Updated version of Tajma using the design system, no lock button and including a collapsible task description (chevron).

Improving Table Accessibility for Screen Readers

After reviewing the organization's accessibility guidelines, I tested Tajma with a visually impaired specialist who relied on auditory and haptic feedback. She described Tajma as unusable. I learned how to use a screen reader and identified several accessibility issues. Through user testing, dialogue with experts, and code adjustments, I made the application usable for visually impaired users and improved accessibility more broadly. Notably, most fixes involved correcting standard code patterns rather than special accommodations.

The older version with a visualization of the screen reader's interpretations, along with navigation issues.

The newer version of Tajma with a visualization of the screen reader's interpretations, along with navigation improvements.

Reflections

Further improvements remain, and I am currently working to conduct a full accessibility evaluation of Tajma. Removing week “delivery” partially solved the edit-ability issue, and I see value in future feedback mechanisms that show users their time is saved automatically.
